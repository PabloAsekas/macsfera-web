Apple<\/strong> ha presentado hoy Developer ID<\/strong>, un sistema que permite a un desarrollador identificar su aplicaci\u00f3n como segura a pesar de que no est\u00e1 en la Mac App Store.<\/strong><\/p>\n

<\/p>\n

Una de las nuevas caracter\u00edsticas de Mountain Lion<\/a><\/strong> es Gatekeeper<\/strong>, un sistema de seguridad que nos permite tener el control de las aplicaciones que podemos instalar en nuestro Mac<\/strong>. De entre sus opciones encontramos una que es: Instalar aplicaciones de la Mac App Store y desarrolladores identificados.<\/strong><\/p>\n

Esta opci\u00f3n permitir\u00e1 instalar apps que est\u00e1n en la tienda de aplicaciones y aquellas que no lo est\u00e1n<\/strong> (por el motivo que sea) pero que si que tienen el Developer ID<\/strong>. De este modo, Apple<\/strong> se asegura que el usuario no puede instalar apps que sean maliciosas<\/strong> y puedan causar un mal funcionamiento en el ordenador o un robo de datos indeseado, permitiendo a los desarrolladores tener aplicaciones que no est\u00e1n en la Mac App Store<\/strong>.<\/p>\n

En la p\u00e1gina de informaci\u00f3n<\/a> de Apple<\/strong> sobre Gatekeeper<\/a><\/strong> vemos un enlace hacia el programa de desarrolladores que ser\u00e1 con el que se pueda obtener el Developer ID<\/strong> para certificar las aplicaciones.<\/p>\n
