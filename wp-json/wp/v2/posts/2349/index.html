Apple<\/a><\/strong> ha contratado otro centro de datos pero esta vez en Santa Clara,<\/strong> una ciudad situada en Silicon Valley, California.
\n
\nEsta jugada puede ser un poco rara ya que hace no mucho decidi\u00f3 comprar 12 petabytes de memoria<\/a> para otro centro de datos lo que nos llev\u00f3 a sospechar mucho m\u00e1s de ese sistema en la nube que se rumorea llamado iCloud<\/a><\/strong>.<\/p>\n

La cosa es que si ten\u00edan uno ya con mucha memoria \u00bfpara qu\u00e9 quieren otro? Porque ese centro de datos no es suyo 100%, pertenece a una empresa llamada DuPont Fabros Technology<\/strong> que les proporciona el servicio.<\/p>\n

El para qu\u00e9 se van a utilizar es muy grande, desde albergar la web oficial, archivos internos, copias de seguridad, dar apollo a los servicios en la nube o como servidor secundario en caso de cat\u00e1strofe natural. El tiempo nos dir\u00e1 para que es este servidor pero yo veo m\u00e1s l\u00f3gico albergar la web oficial o archivos internos ya que la capacidad no es tan grande como la de los otros data centers.<\/p>\n
